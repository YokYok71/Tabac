1. Concepts de base

L'application s'organise autour de deux objets principaux :

  • Tabac — une référence (ex. : Dunhill Early Morning Pipe). Un tabac peut avoir zéro ou plusieurs lots.
  • Lot — une boîte ou quantité précise achetée à une date donnée. C'est le lot qui porte le statut et l'historique de vieillissement.

Cette distinction permet de suivre plusieurs boîtes d'un même tabac à des stades différents (une en cave, une en pot) tout en conservant un seul espace de notes et de description.

2. Cycle de vie d'un tabac

📦 Cave

Un lot nouvellement acheté entre directement en Cave. Il représente une boîte fermée qui vieillit. L'âge est calculé depuis la date de production (si renseignée) ou depuis la date d'achat.

ChampDescription
Poids (g)Quantité en grammes (ex. : 50, 100, 454)
Date d'achatPré-remplie avec la date du jour
Date de productionPour le calcul d'âge précis — reportez-la depuis la boîte
N° boîteNuméro d'identification si vous en avez plusieurs du même tabac
Prix (CHF)Prix d'achat
VendeurBoutique ou revendeur

🫙 Pot

Quand vous ouvrez une boîte pour la mettre en pot, passez le lot en Pot via le bouton ✏️. La date de mise en pot est automatiquement renseignée (modifiable). Le poids peut être mis à jour pour refléter ce qui reste.

✅ Terminé

Quand le lot est vide, passez-le en Terminé. La date de fin est enregistrée automatiquement. Le lot reste visible dans l'historique (bouton Voir terminés) mais n'apparaît plus dans les statistiques actives.

↩️ Réactiver

Un lot terminé peut être réactivé en Cave via le bouton Réactiver. Utile en cas d'erreur ou de boîte retrouvée.

Vous pouvez créer un lot directement en « Pot » ou « Terminé » pour saisir un historique passé.

4. Gérer les lots

Depuis la vue détail d'un tabac (cliquez sur une carte), vous voyez tous ses lots. Bouton + Lot pour ajouter un nouvel achat.

Un même tabac peut avoir plusieurs lots simultanément à des stades différents — par exemple une boîte en cave pour vieillir et un pot en cours de consommation.
  • ✏️ — modifier le statut, le poids, les dates
  • ✕ — supprimer le lot (irréversible)
  • 👁 Voir terminés — afficher l'historique des lots finis

5. Parcourir l'inventaire

Filtres de statut

  • Tous — tabacs avec au moins un lot actif
  • Cave — au moins une boîte en cave
  • Pots — au moins un lot en pot
  • Terminés — au moins un lot terminé

Recherche et tri

La recherche filtre sur nom, marque, composition, catégorie, coupe et notes de dégustation. Tri disponible : Nom · Marque · Note · Âge · Quantité.

Sur le tableau de bord, cliquez sur une catégorie ou une marque pour filtrer directement l'inventaire.

9. Sauvegarde et export

Toutes les options sont dans ⚙️ (icône engrenage, haut à droite).

☁️ Google Drive (recommandé)

Sauvegarde dans un dossier privé de votre Google Drive (invisible dans vos fichiers). Authentification OAuth 2.0 directe avec Google.

  • Sauvegarder Drive — écrase la sauvegarde existante
  • Restaurer Drive — affiche un résumé avant confirmation
La restauration remplace toutes vos données locales. Effectuez un export JSON avant si vous avez un doute.
Sur iOS en mode PWA, l'authentification passe par une redirection — vous quitterez brièvement l'application avant d'y revenir automatiquement.

Autres formats

  • 📋 Export JSON — toutes vos données, réimportable
  • 📄 Export CSV — tabacs, pipes et wishlist (compatible Excel)
  • 📦 Backup ZIP — données + toutes les images
  • 📂 Importer fichier — restaurer depuis un fichier JSON local

1. Core concepts

The app is organized around two main objects:

  • Tobacco — a reference (e.g. Dunhill Early Morning Pipe). A tobacco can have zero or more lots.
  • Lot — a specific tin or quantity purchased on a given date. The lot carries the status and aging history.

This distinction lets you track multiple tins of the same tobacco at different stages (one cellaring, one open in a jar) while sharing a single set of notes and description.

2. Tobacco lifecycle

🛒PurchaseNew lot
→
📦CellarSealed tin
→
🫙JarTin opened
→
✅FinishedLot empty

📦 Cellar

A newly purchased lot enters Cellar status. It represents a sealed tin aging in storage. Age is calculated from the production date (if provided) or the purchase date.

FieldDescription
Weight (g)Quantity in grams (e.g. 50, 100, 454)
Purchase datePre-filled with today's date
Production dateFor accurate age calculation — copy from the tin
Box #ID number if you have several tins of the same tobacco
Price (CHF)Purchase price
SellerShop or vendor

🫙 Jar

When you open a tin and transfer it to a jar, change the lot to Jar status using the ✏️ button. The jar date is auto-filled with today (editable). Update the weight to reflect what remains.

✅ Finished

When the lot is empty, set it to Finished. The end date is recorded automatically. The lot stays in history (via Show finished) but no longer counts in active statistics.

↩️ Reactivate

A finished lot can be moved back to Cellar using the Reactivate button — useful for corrections or rediscovered tins.

You can create a lot directly as "Jar" or "Finished" to record past history.

4. Managing lots

From a tobacco's detail view (tap a card in the inventory), you can see all its lots. Use + Lot to add a new purchase.

A single tobacco can have multiple lots at different stages simultaneously — for example one tin cellaring and one jar being smoked.
  • ✏️ — edit status, weight, dates
  • ✕ — delete lot (irreversible)
  • 👁 Show finished — display the history of finished lots

5. Browsing the inventory

Status filters

  • All — tobaccos with at least one active lot
  • Cellar — at least one tin in the cellar
  • Jars — at least one lot in a jar
  • Finished — at least one finished lot

Search and sort

Search filters on name, brand, blend, category, cut and tasting notes. Sort by: Name · Brand · Rating · Age · Quantity.

On the dashboard, click any category or brand to filter the inventory directly.

9. Backup and export

All options are in ⚙️ (gear icon, top right).

☁️ Google Drive (recommended)

Saves to a private hidden folder in your Google Drive. Direct OAuth 2.0 authentication with Google.

  • Save to Drive — overwrites the existing backup
  • Restore Drive — shows a summary before confirming
Restoring replaces all your local data. Export to JSON first if in doubt.
On iOS PWA, authentication uses a redirect — you will briefly leave the app before returning automatically.

Other formats

  • 📋 Export JSON — all your data, re-importable
  • 📄 Export CSV — tobaccos, pipes and wishlist (Excel-compatible)
  • 📦 Backup ZIP — data + all images
  • 📂 Import file — restore from a local JSON file

10. Updates

⚙️ → Check updates. If a new version is available, save to Google Drive first, then confirm the update.

The app cache is cleared during the update. Back up before continuing.
